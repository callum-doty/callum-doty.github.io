Led end-to-end development of an AI-powered document intelligence system that transformed political design workflows from manual file hunting to semantic discovery. Through four major architectural iterations, reduced infrastructure costs by 75%, improved application performance by 15x, and enabled natural language search across 800+ political campaign files.

The Problem

During the 2024 election cycle, account managers and designers were spending 3-5 minutes per file search navigating our Dropbox file system. With hundreds of daily requests, teams were losing hours of productive time weekly.

Initial analysis focused on search speed, but deeper user research revealed the core issue: creative teams needed to discover thematic inspiration from past campaigns, not just locate files by name. Dropbox's keyword-based search couldn't understand:

  • Campaign messaging strategies and themes
  • Visual design patterns and color palettes
  • Policy focus areas and target demographics
  • Candidate positioning and sentiment

The team needed semantic discovery, not text search.

User Research & Discovery Process

Research Methods

  • • 15+ stakeholder interviews with account managers, designers, and consultants
  • • Process mapping sessions documenting current-state workflows and pain points
  • • 3 rounds of usability testing (v4, v5, final) with iterative refinement
  • • Mental model analysis of how users organize and recall past work

Key Insights

  • ✓ Users remember files by campaign theme and visual style, not filenames
  • ✓ Search criteria: candidate, policy topic, geographic region, message sentiment
  • ✓ Need to discover similar past work for creative inspiration
  • ✓ Existing system requires exact keyword matches - no conceptual understanding

Project Evolution & Learning Journey

V1: Computer Vision Prototype Nov - Dec 2024

Initial approach: ResNet CNN for image classification with Tesseract OCR

Tech stack: Flask, PostgreSQL, TensorFlow, Docker

Learning: Image classification couldn't extract rich metadata (campaign themes, sentiment, target audience). Achieved >95% OCR accuracy but <60% classification accuracy for political content nuances.
V2: LLM Migration Jan - Feb 2025

Pivot: Replaced ResNet with Claude 3 LLM for multi-faceted content analysis

New capabilities: Extract keywords, themes, geographic targeting, sentiment, candidate positioning

Impact: Classification accuracy improved to >90%. Could now extract complex metadata like "healthcare policy focus" and "suburban voter targeting."
V3: Microservices Architecture Mar - May 2025

Scale-up: Added Celery/Redis task queue, MinIO object storage, vector search (pgvector)

Features: Asynchronous processing, semantic search, hierarchical taxonomy, batch operations

Learning: System became over-engineered - 6 containerized services, complex orchestration, ~$84/month infrastructure cost, 30-second startup time. Too heavy for use case.
V4: Optimized Architecture Jun - Aug 2025

Refactor: Migrated to FastAPI, consolidated services, optimized for production

Results: 2-second startup (15x faster), $21/month cost (75% reduction), maintained all features

Success: Production-ready system with right-sized architecture. Proved you can over-optimize - simpler is often better. Deployed to production August 2025.

Hybrid Search Engine

Vector Similarity Search: OpenAI embeddings (1536-dim) stored in pgvector for semantic understanding
Hierarchical Taxonomy: Custom political keyword taxonomy with primary categories → subcategories → canonical terms
Full-Text Search: PostgreSQL tsvector with GIN indexing for keyword-based filtering
Cosine Similarity: Switched from L2 distance to cosine for improved relevance scoring

Multi-Batch AI Pipeline

Batch 1 - Core Metadata: High-priority extraction of document type, summary, strategy (Claude API)
Batch 2 - Classification: Taxonomy mapping, entity extraction, design elements analysis
Batch 3 - Embeddings: Vector generation with contextual enrichment (OpenAI API)
Chain-of-Thought Prompts: Structured reasoning with explicit null handling and strict enumerations

Database Optimization

Eliminated N+1 Queries: SQLAlchemy eager loading with joinedload() for related entities
Strategic Indexing: GIN indexes on search_vector, B-tree on filename and upload_date
Redis Caching: Multi-layer cache for search results, document previews, taxonomy data
Query Optimization: Server-side pagination, filtering pushed to database layer

Async Processing

Celery Task Queue: Background workers for OCR, AI analysis, embedding generation
Rate Limiting: Graceful API throttling with retry logic and exponential backoff
Fault Tolerance: Document recovery mechanism for failed processes with manual retry
Health Monitoring: Status dashboards tracking processing success rates and latency

Senior-Level Engineering Decisions

Prompt Engineering & AI Quality

Developed sophisticated prompt templates with chain-of-thought reasoning to extract political nuances:

  • • Explicit null handling: Force *null* for low-confidence fields to prevent hallucination
  • • Strict enumerations: Fixed-set categorical outputs for reliable filtering
  • • Evidence requirement: LLM must cite source text before generating metadata
  • • Canonical mapping: Two-step "extract then map" to controlled vocabulary
Result: >90% metadata extraction accuracy, 100-point quality scoring system

Document Synthesis for Embeddings

Created holistic vector embeddings by synthesizing multiple content sources:

  • • Structured metadata: Summary, strategy, campaign context from LLM
  • • Full OCR text: Complete extracted content for comprehensive coverage
  • • Taxonomy enrichment: Related hierarchical terms to boost discoverability
  • • Contextual framing: Temporal, geographic, sentiment dimensions
Result: Superior semantic search vs. raw text - finds conceptually similar content

Data Integrity & Deduplication

Implemented comprehensive deduplication strategy to prevent redundant processing:

  • • File hash calculation: SHA-256 hashing for duplicate detection
  • • Lineage tracking: original_document_id to map duplicate relationships
  • • Processing prevention: Skip LLM analysis for known duplicates
  • • Batch recovery: Manual reprocessing for failed documents
Result: Cost savings on API calls, consistent data quality across duplicates

Performance Optimization Journey

Systematic performance improvements through profiling and optimization:

  • • Startup time: 30 seconds → 2 seconds (15x improvement)
  • • Search latency: 60-80% reduction via Redis caching + query optimization
  • • Memory usage: 800MB → 200MB (75% reduction) through service consolidation
  • • Infrastructure cost: $84/month → $21/month via architecture simplification
Learning: Right-sized architecture beats over-engineering. Simplicity scales.

Core Product Features

🔍 Semantic Search

Natural language queries like "healthcare policy graphics for suburban voters" or "Biden campaign positive messaging" return conceptually relevant results using vector similarity and custom political taxonomy.

Technical: Combines pgvector cosine similarity, hierarchical keyword filtering, and full-text search for optimal precision/recall balance.

🤖 Automated Intelligence

AI-powered analysis automatically extracts political themes, policy focus areas, visual design elements, candidate positioning, target demographics, and campaign sentiment from uploaded documents.

Technical: Multi-batch LLM pipeline with chain-of-thought prompts, explicit null handling, and canonical taxonomy mapping.

📊 Admin Dashboard

Minimal monitoring interface with processing metrics, search analytics, quality scores, document review queue, and system health tracking for continuous optimization.

Technical: Real-time metrics via Redis, 100-point quality scoring, manual review workflow for low-scoring documents.

⚡ Background Processing

Asynchronous task queue ensures instant upload response times while comprehensive OCR, AI analysis, and vector embedding generation happen in background workers.

Technical: Celery distributed task queue with Redis broker, retry logic, rate limiting, and fault-tolerant recovery mechanisms.

Key Takeaways & Learnings

Problem Discovery > Feature Building: Initial assumption was "search is slow" but user research revealed the real need: semantic discovery for creative inspiration. Always validate the problem first.

Iterative Architecture: Went through 4 major versions, each teaching valuable lessons. V3's over-engineering proved that complexity isn't always better - V4's simplification delivered better results at 25% the cost.

AI Integration Requires Product Thinking: Raw LLM output isn't enough. Needed custom taxonomy, structured prompts, quality scoring, and user feedback loops to create reliable, production-grade intelligence.

Measure What Matters: Early focus on "time saved" metrics missed the real value - enabling better creative decisions. Shifted to tracking design quality, creative diversity, and strategic inspiration.

"This project taught me that great engineering isn't about using the most advanced tech - it's about deeply understanding the problem, iterating rapidly, and delivering measurable value."
